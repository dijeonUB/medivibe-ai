## 기술 결정 사항

| 결정 | 이유 | 결과 |
|---|---|---|
| `claude-haiku-4-5-20251001` | 빠른 응답 속도, 비용 효율 | 스트리밍 첫 토큰 ~1.5초 |
| 스트리밍 응답 | 대기 체감 시간 단축 | UX 만족도 향상 |
| API Route 서버 사이드 호출 | API 키 클라이언트 노출 방지 | 보안 요구사항 충족 |
| `useRef` 클라이언트 캐시 | 동일 요청 재호출 제거 | 탭 전환 시 즉시 표시 |
| `display:none` 마운트 유지 | 컴포넌트 언마운트 방지 | 백그라운드 프리페치 실현 |
| JSON 강제 System Prompt | 구조화된 응답 파싱 안정화 | 파싱 오류율 감소 |

## 발생한 문제 및 해결

| 문제 | 원인 | 해결 |
|---|---|---|
| 건기식 JSON 파싱 오류 | max_tokens 부족으로 JSON 중간 절단 | max_tokens 2048 → 3500으로 증가 |
| 탭 전환 시 캐시 초기화 | `&&` 조건부 렌더링으로 컴포넌트 언마운트 | `display:none` 방식으로 전환 |
| 건강기록 검색 시 캘린더 사라짐 | 조건부 표시 로직 | 2패널 고정 레이아웃으로 재설계 |
| 챗봇 이미지 저화질 | PNG 썸네일 사용 | 고화질 JPG로 교체 |
