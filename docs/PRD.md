# PRD — MediVibe AI 제품 요구사항 정의서

## 1. 제품 개요

**제품명:** MediVibe AI
**목적:** 일반인이 의료 용어와 건강 정보를 쉽게 이해할 수 있도록 Claude AI가 설명해주는 웹 서비스
**대상 사용자:** 유비케어 플랫폼 이용자, 의료 정보에 관심 있는 일반인
**버전:** v1.3.0 (2026-03-13)

---

## 2. 배경 및 문제 정의

| 문제 | 현황 | 영향 |
|---|---|---|
| 의료 용어 장벽 | 처방전·검사결과의 전문 용어를 환자가 이해 불가 | 의료 불신, 복약 오류 |
| 정보 과부하 | 포털 검색 결과 방대, 신뢰도 판별 어려움 | 의료 정보 불평등 |
| 단절된 건강 관리 | 상담 내역 분산, 연속적 관리 불가 | 재진 시 맥락 손실 |
| 맥락 없는 영양제 추천 | 쇼핑몰 리뷰 기반, 개인 상황 미반영 | 불필요한 지출·부작용 |

---

## 3. 핵심 기능 요구사항

### 기능 요구사항
| 기능 | 설명 | 우선순위 | 구현 상태 |
|---|---|---|---|
| AI 건강상담 채팅 | Claude API 기반 의료 Q&A, 스트리밍 응답 | P0 | ✅ 완료 |
| 증상 체크 | 증상 입력 → 추정 진료과·주의사항 반환 | P0 | ✅ 완료 |
| 건강기록 캘린더 | 상담 이력 날짜별 시각화, 검색 기능 | P1 | ✅ 완료 |
| 건강기능식품 추천 | 카테고리·정렬 기반 AI 맞춤 추천 10개 | P1 | ✅ 완료 |
| 건강뉴스 | 현재 계절 기반 AI 건강 기사 8건 | P1 | ✅ 완료 |
| 멀티턴 대화 | 대화 맥락 유지, 이어서 상담 | P1 | ✅ 완료 |
| 면책 고지 | 의료 조언 아님 안내 | P0 | ✅ 완료 |

### 비기능 요구사항
- 응답 시간: 첫 토큰 2초 이내 (스트리밍)
- 캐시 전략: 동일 요청 재호출 방지 (`useRef` 클라이언트 캐시)
- 모바일·PC 반응형 UI (Tailwind 반응형 그리드)
- 한국어 전용 서비스

## 4. 기술 스택

| 영역 | 기술 | 선택 이유 |
|---|---|---|
| Frontend | Next.js 16.1 (App Router), TypeScript | SSR + API Route 통합, 타입 안전성 |
| Styling | Tailwind CSS 4 | 빠른 UI 개발, 반응형 지원 |
| AI | Anthropic Claude API (claude-haiku-4-5-20251001) | 빠른 응답, 한국어 품질, 비용 효율 |
| Backend | Next.js API Routes (서버리스) | API 키 서버 사이드 보호 |
| 배포 | Vercel | Next.js 최적화 플랫폼, 무중단 배포 |

---

## 5. 경쟁 차별화

| 비교 항목 | 기존 의료 챗봇 | MediVibe AI |
|---|---|---|
| 정보 범위 | 단순 FAQ 또는 증상 검색 | 상담·증상·기록·영양제·뉴스 통합 |
| AI 모델 | 룰 기반 또는 범용 LLM | Claude API (의료 컨텍스트 프롬프트 설계) |
| 응답 방식 | 정적 응답 | 실시간 스트리밍 타이핑 |
| 개인화 | 없음 | 건강기록 캘린더 + 이어서 상담 |
| 계절 연동 | 없음 | 현재 월·계절 기반 건강정보 자동 생성 |
| 유비케어 연계 | 해당 없음 | EMR/HIS 연동 확장 가능한 아키텍처 |

---

## 6. 제약 조건

- API 키는 서버 사이드에서만 사용 (보안)
- 의료 진단·처방 기능 제공 불가 (법적 제약 — 의료법 제27조)
- 응답은 한국어로만 제공
