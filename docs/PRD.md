# PRD — MediVibe AI 제품 요구사항 정의서

## 1. 제품 개요

**제품명:** MediVibe AI
**목적:** 일반인이 의료 용어와 건강 정보를 쉽게 이해할 수 있도록 Claude AI가 설명해주는 웹 서비스
**대상 사용자:** 유비케어 플랫폼 이용자, 의료 정보에 관심 있는 일반인

## 2. 배경 및 문제 정의

- 의료 용어는 전문적이어서 환자/보호자가 이해하기 어렵다
- 검색 결과는 너무 많고 복잡하며 신뢰도 판단이 어렵다
- 유비케어는 EMR/HIS 솔루션 기업으로 의료 정보 서비스 역량 보유

## 3. 핵심 기능 요구사항

### 기능 요구사항
| 기능 | 설명 | 우선순위 |
|---|---|---|
| AI 채팅 | Claude API 기반 의료 Q&A | P0 |
| 스트리밍 응답 | 실시간 타이핑 효과 | P0 |
| 빠른 질문 | 예시 질문 버튼 제공 | P1 |
| 멀티턴 대화 | 대화 맥락 유지 | P1 |
| 면책 고지 | 의료 조언 아님 안내 | P0 |

### 비기능 요구사항
- 응답 시간: 첫 토큰 2초 이내
- 모바일 반응형 UI
- 한국어 전용 서비스

## 4. 기술 스택

- **Frontend:** Next.js 14 (App Router), TypeScript, Tailwind CSS
- **Backend:** Next.js API Routes (서버리스)
- **AI:** Anthropic Claude API (claude-haiku-4-5-20251001)
- **배포:** Vercel

## 5. 제약 조건

- API 키는 서버 사이드에서만 사용 (보안)
- 의료 진단/처방 기능 제공 불가 (법적 제약)
- 응답은 한국어로만 제공

## 6. 성공 지표

- 앱 정상 동작 (배포 URL 접근 가능)
- Claude API 스트리밍 응답 구현
- AI-Native 개발 프로세스 문서화 완료
